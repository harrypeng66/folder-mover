# 素材图名清洗助手

这是一个带 GUI 的本地 Python 工具，用来递归扫描素材目录，统一清洗图片文件名里的冗余尾缀，并把处理后的结果输出到新的中文目录中。

软件界面内置开发者署名：

`需求及bug提交｜项目合作｜微信：tktk6622`

## 功能

- 选择任意素材文件夹，递归处理其下全部子文件夹和图片
- 扫描前先统计图片总数、子目录数，以及预计会改名的图片数量
- 点击 `处理` 后，一键复制到新的中文结果目录，不改动原始素材
- 默认支持清理常见尾缀：`-1`、`-2`、`(1)`、`-(1)`
- 支持自定义正则表达式
- 支持多个正则，通过 `|` 连接
- 处理过程中显示实时进度条和日志
- 如果清洗后出现重名，自动追加 `_重名01`、`_重名02` 之类的后缀避让

## 处理逻辑

示例文件名：

- `A100-1.jpg` -> `A100.jpg`
- `A100-2.png` -> `A100.png`
- `2603200012-(1).jpg` -> `2603200012.jpg`
- `面料图(1).webp` -> `面料图.webp`

程序只会从扩展名前开始，向左匹配并清除尾部规则，不会直接删掉文件名中间的正常内容。

处理结果会输出到素材目录同级的新文件夹中，例如：

`原目录名_处理结果`

如果同名目录已存在，会自动生成：

`原目录名_处理结果_01`

## 运行

```bash
python3 material_image_name_cleaner.py
```

## 打包

Windows 本地打包脚本：

```text
build_windows.bat
```

GitHub Actions 云端打包工作流：

```text
.github/workflows/build-exe.yml
```

构建产物：

- `dist/material_image_name_cleaner.exe`
- `material_image_name_cleaner-win.zip`

## 依赖

当前项目不依赖第三方运行库，GUI 使用标准库 `tkinter`。

打包时会额外安装：

- `pyinstaller`

## 注意

- 如果本机 Python 缺少 `tkinter`，本地无法直接打开 GUI，但不影响云端打包 Windows EXE。
- 正则表达式会被强制约束在文件名尾部生效，适合做尾缀清洗。
